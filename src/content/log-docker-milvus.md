# Docker 容器化部署与 Milvus 向量检索实践

在构建高性能的人脸搜索服务时，环境的一致性与检索速度是两个关键挑战。本文记录了如何利用 Docker 进行服务容器化部署，并结合 Milvus 向量数据库实现毫秒级的人脸模糊匹配。

## Docker 容器化部署

为了解决“在我电脑上能跑，在服务器上跑不起来”的经典问题，我将整个后端服务进行了 Docker 化。

### 1. Dockerfile 编写
- **基础镜像**：选择轻量级的 `python:3.9-slim`。
- **依赖安装**：通过 `requirements.txt` 安装 Flask, pymilvus, opencv-python 等核心库。
- **环境配置**：设置时区、工作目录及必要的环境变量。

### 2. Docker Compose 编排
使用 `docker-compose.yml` 一键启动整个服务栈：
- **Web 服务**：后端 API 服务。
- **Milvus**：向量数据库服务（包含 etcd, minio 等依赖）。
- **Nginx**：反向代理与静态资源托管。

## Milvus 向量检索：人脸查询

传统的数据库无法高效处理非结构化数据（如图片），而向量数据库是解决这一问题的神器。

### 1. 向量提取
使用 `face_recognition` 或 `InsightFace` 模型将人脸图片转化为 128 维或 512 维的特征向量。这些向量代表了人脸的唯一特征。

### 2. Milvus 存储与索引
- **建库**：在 Milvus 中创建 Collection，定义向量字段维度。
- **索引**：构建 `IVF_FLAT` 或 `HNSW` 索引，加速检索速度。

### 3. 模糊匹配查询
当用户上传一张照片时：
1.  提取该照片的人脸向量。
2.  在 Milvus 中进行近似最近邻搜索 (ANN)。
3.  设置相似度阈值（如欧氏距离 < 0.6），返回最相似的人脸 ID。

通过这种架构，我们实现了亿级数据规模下的毫秒级人脸搜索，且部署过程极其丝滑。
