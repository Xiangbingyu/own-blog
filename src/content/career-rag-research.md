# RAG 技术深度探索：从基础索引到 GraphRAG

在开发 AI 应用的过程中，检索增强生成 (RAG) 是解决大模型幻觉与知识时效性的关键技术。我深入研究了多种 RAG 范式，并在实际项目中进行了落地实践。

## RAG 优化策略体系

我总结并实践了 16 种 RAG 优化方向，重点包括：

1.  **语义分块与上下文增强**：不只是简单的固定长度切分，而是根据语义完整性进行切块，并为每个切块附带上下文或自动生成的总结标题，显著提升检索的召回准确率。
2.  **HyDE (Hypothetical Document Embeddings)**：针对查询问题，先让 LLM 生成一个“假设性答案”，再对该答案进行向量化检索。这在跨领域或低频问题的检索中效果显著。
3.  **融合检索 (Hybrid Search)**：结合 **向量检索 (FAISS/Milvus)** 的语义匹配能力与 **关键词检索 (Elasticsearch/BM25)** 的精确匹配能力，通过加权排序（Rerank）输出最终结果。
4.  **多重检索 (Multi-query Retriever)**：利用 LangChain 将用户的一个复杂问题拆解为多个子问题并行检索，提高信息覆盖度。

## GraphRAG：结构化知识的未来

针对传统 RAG 难以处理的“全局性问题”或“跨文档推理”，我深入研究了 **GraphRAG** 技术：
- **原理**：先对文本分块，利用 LLM 提取“主体-行为-客体”关系，构建知识图谱。
- **抽象**：对图谱进行社区检测（Community Detection）与层级摘要，生成高维度的核心结论。
- **应用**：在检索时，不仅匹配向量，还能沿着图谱关系链进行多跳推理，极大地提升了复杂问答的深度与逻辑性。
