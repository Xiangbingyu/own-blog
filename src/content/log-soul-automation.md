# Soul APP 比赛复盘：自动化脚本在数据增长中的应用

在参加 Soul APP Agent 创新赛期间，为了测试 Agent 在高并发互动下的表现（当然也为了提升一点人气），我编写了一套自动化脚本来模拟用户点赞与交互。

## 技术实现

### 1. 协议分析与模拟
通过抓包工具（Charles/Fiddler）分析了 Soul APP 的点赞接口。发现其签名算法虽然复杂，但并非不可破解。
- **请求重放**：构造了包含特定 Header 和 Payload 的 HTTP 请求。
- **设备指纹**：为了避免被风控，构建了一个设备指纹库，每次请求模拟不同的 Device ID 和 User Agent。

### 2. 自动化执行
使用 Python 编写了多线程脚本：
- **代理池**：接入了商业代理 IP，确保每个请求来自不同的 IP 地址。
- **任务调度**：实现了简单的生产者-消费者模型，控制点赞频率，模拟真实用户的操作曲线。

## 反思与总结
虽然通过脚本实现了数据的短期增长，但这次经历让我更深刻地理解了平台的反作弊机制。在比赛后期，官方的风控策略升级，导致部分脚本失效。这也反向促进了我对网络安全与风控策略的学习。

> *注：技术本身无对错，但应在规则允许的范围内使用。本次实践仅作为技术探索与压力测试。*
